静态代理与动态代理

30 Oct 2019, 11:42
设计模式
代理模式

代理模式：

​ 代理是当两个类需要通信时，引入第三方代理类，将两个类的关系解耦，让我们只了解代理类即可，而且代理的出现还可以让我们完成与另一个类之间的关系的统一管理，但是切记，代理类和委托类要实现相同的接口，因为代理真正调用的还是委托类的方法。通俗的讲 现在的明星经纪人就是类似代理类的作用

/**
 * 被代理对象  必须实现某个接口
 * @author lennon
 */
public class Lennon implements Study {
    @Override
    public void study() {
        System.out.println("今天我学了动态代理");
    }
}

/**
 * @author lennon
 */
public interface Study {
    /**
     * 学习
     */
    void study();
}

/**
 *  静态代理：
 *  这种代理方式需要代理对象和被代理对象实现一样的接口，可以在不修改目标对象的前提下扩展目标对象的功能。
 *  缺点：静态代理只能为一个目标对象服务，如果目标对象过多，会产生过多的代理类。不易维护，一旦接口增加方
 *  法，目标对象与代理对象都要进行修改。
 * @author lennon
 */
public class StaticProxy implements Study {
	// 被代理对象
    private Lennon lennon;
    StaticProxy(Lennon lennon){
        this.lennon = lennon;
    }

    /*
     * 如果只想代理类服务自己
     *      StaticProxy(){
     *         this.lennon = new Lennon();
     *      }
     */

    /**
     * 增加升级方法
     */
    private void up(){
        System.out.println("我升级了");
    }

    @Override
    public void study() {
        lennon.study();
        up();
    }
}



/**
 *  静态代理测试类
 * @author lennon
 */
public class ProxyTest {
    public static void main(String[] args) {
        // 被代理对象
        Lennon lennon = new Lennon();
        StaticProxy staticProxy = new StaticProxy(lennon);
        staticProxy.study();
        /*
        输出：  今天我学了动态代理
                我升级了
         */
    }
}

/**
 * 动态代理类：动态代理对象不需要实现接口，但是要求目标对象必须实现接口，否则不能使用动态代理
 * @author lennon
 */
public class DynamicProxy {
    public static void main(String[] args) {
        // 拿到被代理对象
        Lennon lennon = new Lennon();
         Study proxyLennon = (Study) Proxy.newProxyInstance(lennon.getClass().getClassLoader(), lennon.getClass().getInterfaces(), (proxy, method, args1) -> {
            // 如果执行了学习方法就输出我升级了
            if ("study".equals(method.getName())){
                // 执行目标对象方法
                method.invoke(lennon,args1);
                System.out.println("我今天又升级了");
            }else {
                // 如果不是执行了study方法 就执行原方法
                return method.invoke(lennon,args1);
            }
            return null;
        });
         // 代理类调用方法
         proxyLennon.study();
        /*
        输出：   今天我学了动态代理
				我今天又升级了
         */
    }
}

静态代理与动态代理的区别主要在：

  • 静态代理在编译时就已经实现，编译完成后代理类是一个实际的class文件
  • 动态代理是在运行时动态生成的，即编译完成后没有实际的class文件，而是在运行时动态生成类字节码，并加载到JVM中

扩展：

​ cglib代理：是一个第三方代码生成类库，运行时在内存中动态生成一个子类对象从而实现对目标对象功能的扩展。如果想代理没有实现接口的类，就可以使用CGLIB实现。
